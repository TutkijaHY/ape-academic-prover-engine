If one source dominates (HHI ≥ 0.50), Self-RAG triggers re-retrieval. APE ranks by relevance and epistemic diversity — not relevance alone. Three-Tier Source Typing [S]/[R]/[D] APE separates the corpus into three layers: [S] Scholarly articles, [R] Own research texts, and [D] Numerical data. Each layer uses different chunking, retrieval weighting, and diversity calculation — answers are drawn from the right type of evidence. Unified Multimodal Index APE indexes both text and images from PDF documents. A Vision Language Model generates descriptions of images and figures; tables are parsed at structural level using IBM Granite-Docling (TEDS-structure 0.97, IBM-reported). One query retrieves prose, tables, and visual evidence from the same index. Configurable Self-RAG Quality Gate Before delivering an answer, APE checks three criteria: enough sources found, no single source dominating ([S]/[R]/[D] balance maintained), and source diversity within acceptable limits. If any criterion fails, retrieval runs again. All thresholds set in .env. Model-Agnostic by Design APE has no hardcoded LLM model names anywhere in the codebase. The active model is selected from a dropdown list of locally installed Ollama models. Switching to a new LLM requires no code changes — token limits estimated dynamically. A built-in comparison view runs the same query across all locally installed models simultaneously. 100% Local Processing All processing runs on the local machine — no cloud dependencies, no API keys, no data transmitted externally.
